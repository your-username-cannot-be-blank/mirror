Не совсем. РКН заблокировал только matrix-client.matrix.org, но не matrix.org. matrix.org доступен. Но большинство клиентов, включая официальный Element и app.element.io (веб версия) следуют рекомендации сервера и используют заблокированный matrix-client.matrix.org. Также хочу обратить внимание, что из-за изменения в серверной политике в старых клиентах не работает просмотр и скачивание файлов.

К слову, рекомендация использовать matrix-client.matrix.org вместо matrix.org приходит отсюда https://matrix.org/.well-known/matrix/client
Ответ сервера:

{
    "m.homeserver": {
        "base_url": "https://matrix-client.matrix.org"
    },
    "m.identity_server": {
        "base_url": "https://vector.im"
    },
    "org.matrix.msc4143.rtc_foci": [
        {
            "type": "livekit",
            "livekit_service_url": "https://livekit-jwt.call.matrix.org"
        }
    ]
}

А вот в NeoChat штатно нельзя поменять, но можно подредактировать файл ~/.config/KDE/neochat.conf после авторизации. Т.е. первый вход должен быть под VPN/прокси, а потом можно настроить напрямую (заменив matrix-client.matrix.org на matrix.org). Если у вас учётка там. Возможно, так можно подправить и другие клиенты.
Старый Fractal 4.4.2 (GTK3 based) сохраняет здесь ~/.local/share/keyrings/Default_keyring.keyring

Можно, конечно, зарегиться на другом сервере, но мне кажется, они менее надёжные и федерация иногда глючит. Но на смартфоне так легко сервер не поменять.

Что делает команда /join в Toxic?
Команда /join используется в Toxic для присоединения к постоянной конференции (персистентной группе). Этот код — уникальный идентификатор группы (Group ID), и по нему можно подключиться.

Toxic — единственный клиент, который напрямую поддерживает такую систему команд.

Чем отличаются группы в Toxic и других клиентах?

  • Toxic поддерживает персистентные группы с постоянными идентификаторами, к которым можно присоединиться в любое время, если у тебя есть Group ID.

  • В qTox и uTox группы работают иначе:

    • Групповой чат создаётся временно — участники остаются подключёнными, пока активен создатель.
    • Нет функции присоединиться к группе по Group ID.

Как создать или подключиться к группе в qTox и uTox?
В этих клиентах нет команды /join. Чтобы вступить в группу:

  1. Создание группы:
    • В qTox нажми на кнопку с изображением группы и выбери “Create group chat”.
    • В uTox — нажми правой кнопкой по контакту и выбери “Start group chat”.
  2. Добавление участников: Создатель может пригласить тебя, нажав “Invite friend” в групповом окне.

Важно: Эти группы будут существовать, только пока активен их создатель.

Можно ли использовать Group ID в других клиентах?
Нет, qTox и uTox не поддерживают Group ID, как в Toxic. Чтобы присоединиться, тебе нужно:

  • Либо попросить кого-то из участников пригласить тебя.
  • Либо использовать Toxic для работы с /join.

Если хочешь быть в этой группе постоянно — Toxic будет лучшим выбором.

Традиционные группы (qTox, uTox):

  • Не требуют серверов — общение идёт напрямую между участниками.
  • Временные по своей природе — не сохраняются, если все участники выйдут.

Если создатель группы выходит:

  • Группа не исчезает, если в ней остаются хотя бы два активных участника.
  • Если все участники выйдут (включая создателя), группа будет удалена, и восстановить её нельзя.

Что происходит при сетевых проблемах?

  • Если у создателя проблемы с сетью, группа не пропадёт, если другие участники остаются онлайн.
  • Однако, если ты перезагрузишь клиент или потеряешь соединение, группа исчезнет из интерфейса.

Постоянные группы (Toxic):

  • Имеют Group ID, позволяющий вступать в группу позже.
  • Не зависят от создателя — любой участник может вернуться по ID, даже если создатель офлайн.

Если все выйдут из группы:

  • Группа остаётся доступной, пока кто-то знает Group ID.
