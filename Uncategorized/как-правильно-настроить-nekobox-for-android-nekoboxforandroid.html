skripterasm(Roman)

Можно без картинок настроить, представлю ниже минимальные изменения в настройках, чтобы через прокси ломились только выбранные приложения и все это работало. Это позволит не дергать туда-сюда проксю во время использования телефона, разве что андроид может выгружать из памяти периодически сам некобокс.

Это будет: экономить трафик, меньше палить прокси-сервер, а так же банковские приложения не будут жаловаться на включенный ВПН.

Настройки
Раздел: Настройки приложения
Сервисный режим - выбираем VPN

Раздел: Настройки маршрута
Режим VPN для приложений - ВКЛ (Внутри этого пункта выбираем режим “Прокси” и активируем те приложения, которые нужно направлять через прокси)

Маршруты
Чтобы использовать маршруты в режиме когда приложения проксируются по “белому списку”, а не вся система нужно чтобы приложение для которой рисуется маршрут было в том списке.

Например чтобы браузер мог попасть на этот сайт нужно добавить его настройках как написал выше, после чего он будет подчиняться кастомным маршрутам.

При добавлении маршрута нужно прописать минимум:

  1. название
  2. domain (например ntc.party)
  3. outbound - прокси

Остальные параметры это дополнительные фильтры по приложению и сетевым параметрам.

Xunlei

Доходчиво навряд ли найду. Два способа это сделать: запустить клиент тунеля на VPS и настроить транзит, либо запустить клиент тунеля на конечном устройстве, забиндив Outbound к виртуальному адаптеру уже настроенного тунеля к VPS или добавив статический маршрут для этого подключения в таблицу маршрутизации через шлюз настроенного тунеля к VPS. В первом случае будет менее безопасно, т.к. пакеты будут расшифроваться на VPS и видны, во втором случае будет снижен payload из-за уменьшения MTU вложенного тунеля. Для ваших целей видимой разницы не будет.

Самый простой вариант запустить бесплатный прокси через уже настроенный прокси — взять игру в EGS можно будет.
