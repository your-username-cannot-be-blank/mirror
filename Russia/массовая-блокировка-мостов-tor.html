There is documentation on the Tor issue tracker of obfs4 bridges being blocked, but only in mobile networks. Does that match your experience?

GitLab

[Russia] Blocking Tor resources (2024 edition) (#40046) · Issues · The Tor...

This ticket documents the ongoing censorship in Russia targeting Tor in 2024. Below some issues related to this censorship:

Mobile connectivity issue: Russian users reported that the same obfs4 bridge works on their desktop (with a residential ISP), but doesn’t work on their mobile device (using a mobile ISP). The most frequently mentioned mobile ISP is Tele2 (owned by Rostelecom, Russia’s state-owned ISP). Users also mentioned issues with Beeline and Yota. Yota uses Megafon network, but neither Megafon nor MTS, nor smaller mobile providers like Sber or Tinkoff mobile were mentioned. Some users reported that obfs4 bridges seems to be affected in 4G mobile networks. One user shared that when they switched from 4G to 3G on their mobile phone - the bridges that did not work started working again.

GitLab

[Russia] Investigate obsf4 blocking by protocol in mobile 4G in some ISPs...

Since August 2024, we've seen a rise in user support tickets from Russia. Users reported issues with bridges provided by GetBridgesBot, noting that these bridges didn't connect/work for...

Since August 2024, we’ve seen a rise in user support tickets from Russia. Users reported issues with bridges provided by GetBridgesBot, noting that these bridges didn’t connect/work for them.

Initially, we suspected that some bridges distributed via Telegram had been enumerated and blocked. However, more recent findings from @nina (full report: [Russia] Blocking Tor resources (2024 edition) (#40046) · Issues · The Tor Project / Anti-censorship / censorship-analysis · GitLab) suggest that obfs4 is blocked in mobile/4G network in some ISPs in Russia.

  • The same obfs4 bridge connects successfully on residential connections, but fails on mobile networks (4G).
  • Downgrading to 3G allows these bridges to connect to Tor without issues.
  • WebTunnel bridges remain unaffected.

Just yesterday, a user reported that obfs4 bridges were not working on MTS/Moscow (AS8359).
I checked the IPs of these bridges using MTS Looking Glass, and they were reachable, which could be an evidence of a new type of block (by protocol and not by IP).

Providers mentioned by users: Tele2 (N1), Beeline (N2), and MTS.
